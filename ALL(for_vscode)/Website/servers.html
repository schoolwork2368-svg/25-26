Frontend & Backend

Web development is broadly divided into two areas: the frontend (client-side) and the backend (server-side). They work together to deliver a complete web application.

Frontend (Client-Side)

The frontend is everything the user sees and interacts with in their browser. It's responsible for the user interface (UI) and user experience (UX).

Core technologies include:

  • HTML (HyperText Markup Language): The standard markup language for documents designed to be displayed in a web browser. It provides the structure of the page.
  • CSS (Cascading Style Sheets): A style sheet language used for describing the presentation of a document written in HTML. It controls the layout, colors, and fonts.
  • JavaScript: A programming language that enables interactive web pages. It is used for things like form validation, animations, and fetching data from the backend without reloading the page.

Backend (Server-Side)

The backend is the part of the application that the user doesn't see. It runs on a server and is responsible for storing and organizing data, and ensuring everything on the client-side works. It communicates with the frontend, sending and receiving information to be displayed.

Common backend tasks include database operations, user authentication, and business logic. Popular backend languages and environments include:

  • Node.js (with JavaScript)
  • Python (with Django or Flask)
  • PHP
  • Ruby (with Ruby on Rails)

Web Servers

A web server is software that serves web pages to users upon request. It handles incoming HTTP requests from clients (usually web browsers) and responds with the requested resources, such as HTML files, images, or other content.

Popular Web Servers

Some of the most widely used web servers include:

  • Apache HTTP Server: One of the oldest and most reliable web servers, known for its flexibility and wide range of features.
  • Nginx: A high-performance web server and reverse proxy server known for its speed and ability to handle a large number of concurrent connections.
  • Microsoft Internet Information Services (IIS): A web server developed by Microsoft, commonly used in Windows Server environments.
  • Caddy: A modern web server with automatic HTTPS configuration and easy setup.

Functionality

Web servers can also provide additional functionalities such as load balancing, caching, URL rewriting, and security features like SSL/TLS encryption.

They play a crucial role in the overall architecture of web applications, ensuring that content is delivered efficiently and securely to users around the world.

Common Backends Servers

In addition to web servers, there are various backend servers that handle specific tasks in web development. Some common types include:

Application Servers

Application servers host and run web applications, providing the necessary environment for executing server-side code. Examples include:

  • Node.js: A JavaScript runtime built on Chrome's V8 engine, commonly used for building scalable network applications.
  • Tomcat: An open-source Java Servlet Container developed by the Apache Software Foundation.
  • WildFly: A flexible, lightweight application server for Java applications.

Database Servers

Database servers store and manage data for web applications. They respond to queries from application servers and provide data retrieval and storage functionalities. Examples include:

  • MySQL: An open-source relational database management system.
  • PostgreSQL: An advanced open-source relational database known for its robustness and extensibility.
  • MongoDB: A popular NoSQL database that stores data in flexible, JSON-like documents.

APIs and Servers

APIs (Application Programming Interfaces) are sets of rules and protocols that allow different software applications to communicate with each other. In web development, APIs often interact with backend servers to fetch or manipulate data.

How APIs Work with Servers

When a client (like a web browser or mobile app) makes a request to an API, the request is sent to a backend server. The server processes the request, interacts with databases or other services if necessary, and then sends back a response to the client.

Types of APIs

Common types of APIs used in web development include:

  • RESTful APIs: Use standard HTTP methods (GET, POST, PUT, DELETE) and are stateless.
  • GraphQL APIs: Allow clients to request only the data they need, reducing over-fetching and under-fetching of data.
  • SOAP APIs: A protocol for exchanging structured information in web services using XML.

Common Banckend Servers

Here are some common backend servers used in web development:

  • Node.js: A JavaScript runtime built on Chrome's V8 engine, widely used for building scalable network applications.
  • Express.js: A minimal and flexible Node.js web application framework that provides a robust set of features for web and mobile applications.
  • Django: A high-level Python web framework that encourages rapid development and clean, pragmatic design.
  • Flask: A lightweight WSGI web application framework in Python, known for its simplicity and flexibility.
  • Ruby on Rails: A server-side web application framework written in Ruby, following the model-view-controller (MVC) architecture.
  • Laravel: A PHP web application framework with expressive, elegant syntax, designed for ease of use and rapid development.
  • Spring Boot: A Java-based framework used to create stand-alone, production-grade Spring-based applications with minimal configuration.
  • Firebase: A platform developed by Google for creating mobile and web applications, providing backend services like real-time databases, authentication, and hosting.
  • ASP.NET Core: A cross-platform, high-performance framework for building modern, cloud-based, internet-connected applications using .NET.
  • Apache Tomcat: An open-source implementation of the Java Servlet, JavaServer Pages, and Java Expression Language technologies.
  • Supabase: An open-source Firebase alternative that provides a backend-as-a-service platform with a PostgreSQL database, authentication, and real-time capabilities.
  • GraphQL Servers: Servers that implement the GraphQL query language, allowing clients to request only the data they need. Examples include Apollo Server and Hasura.
